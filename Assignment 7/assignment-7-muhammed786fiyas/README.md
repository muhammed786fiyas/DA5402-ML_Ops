## Running the Project

### Option A — Run via MLflow Project (Recommended)
```bash
mlflow run . -P learning_rate=0.001 -P batch_size=64 -P epochs=5 \
  --experiment-name MNIST-Classifier \
  --env-manager=local
```

### Option B — Run Training Script Directly
```bash
python src/train.py --learning_rate 0.001 --batch_size 64 --epochs 5
```

**Available parameters:**

| Parameter | Type | Default | Description |
|---|---|---|---|
| `learning_rate` | float | 0.001 | Adam optimizer learning rate |
| `batch_size` | int | 64 | Training batch size |
| `epochs` | int | 5 | Number of training epochs |

## Model Serving

Once you have identified the best run from the MLflow UI, serve the model:

```bash
mlflow models serve \
  -m "runs:/<run_id>/model" \
  --port 5001 \
  --env-manager=local \
  --no-conda
```

Replace `<run_id>` with the Run ID of your best experiment run.

The model is now available at `http://127.0.0.1:5001/invocations`.

## Streamlit Web Client

In a separate terminal (keep the MLflow server running):

```bash
streamlit run app/app.py
```

Open `http://localhost:8501` in your browser.

**How to use:**
1. Upload a grayscale handwritten digit image (PNG/JPG)
2. The app sends the image to the MLflow REST API
3. The predicted digit and confidence score are displayed
4. A probability distribution chart shows scores for all 10 digits

> **Note:** The model is trained on MNIST-style images. For best results, use images exported from the MNIST dataset.

### Generate MNIST Test Images
```bash
python -c "
import torchvision
dataset = torchvision.datasets.MNIST(root='data', train=False, download=False)
for i in range(10):
    img, label = dataset[i]
    img.save(f'test_digit_{label}.png')
    print(f'Saved test_digit_{label}.png — Label: {label}')
"
```

## Model Architecture

A 2-layer CNN classifier:

```
Input (1, 28, 28)
    → Conv2d(32) + ReLU + MaxPool → (32, 14, 14)
    → Conv2d(64) + ReLU + MaxPool → (64, 7, 7)
    → Flatten → Linear(3136→128) + ReLU + Dropout(0.3)
    → Linear(128→10)
    → Output: 10 class logits
```

**Best run results:**

| Metric | Value |
|---|---|
| Val Accuracy | 99.05% |
| Val Loss | 0.0301 |
| Learning Rate | 0.001 |
| Batch Size | 128 |
| Epochs | 5 |

---

## Experiment Comparison

Three hyperparameter configurations were evaluated:

| Run | Learning Rate | Batch Size | Val Accuracy |
|---|---|---|---|
| lr0.001_bs64_ep5 | 0.001 | 64 | 98.92% |
| lr0.001_bs128_ep5 | 0.001 | 128 | **99.05%** |
| lr0.01_bs64_ep5 | 0.01 | 64 | 11.16% |

**Conclusion:** `lr=0.001, batch_size=128` gave the best results. A learning rate of `0.01` caused training instability.
